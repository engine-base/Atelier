# T-F-31 CI Gate #12 — selected-stack カバレッジ検証

## Purpose

`03_architecture/selected-stack.json` で確定された技術スタックの各項目が、
実装側 (`package.json` deps / `apps/api/pyproject.toml` deps / config ファイル等) で
実際に配線されていることを CI で機械的に検証する。

これにより「selected-stack に書いたが代替で動かしている」「placeholder のまま忘れた」
を構造的に防ぐ。

## Behavior

1. `03_architecture/selected-stack.json` を読み込み、各 selection (`chosen` 値) を抽出
2. 検証カテゴリごとに対応する実装ファイルを check:
   - **deps 系** (uv, pnpm, anthropic SDK, fastapi, sqlalchemy, drizzle 等): `package.json` / `pyproject.toml` の dependencies に含まれるか
   - **lint/format 系** (ruff, ESLint, Prettier, shellcheck): `pyproject.toml [tool.ruff]` / `.eslintrc.json` / `.prettierrc` / `.github/workflows` の step 存在
   - **CI 系** (GitHub Actions, Husky, pre-commit): `.github/workflows/*.yml` / `.husky/*` の存在
   - **hosting 系** (Vercel, Fly.io, Supabase): config ファイル存在 (`vercel.json` / `fly.toml` / `supabase/config.toml`)
3. 未配線項目があれば **FAIL** + その項目を error 出力

## Skippable Categories

以下のカテゴリは「将来 task で配線予定」として exempt list で除外できる:
- `deferred_decisions[]` に列挙された項目
- 明示的に `tier_3_regression` のいずれかの gate が WIP の項目

exempt list は `03_architecture/selected-stack.json#deferred_decisions` を
信頼源とする (CI 側では追加しない)。

## Implementation Note

`v3-gate.yml` に `gate-12-selected-stack-coverage` job を追加。inline python heredoc
で実装。マッピング table は職別 (deps / lint / CI / hosting) で分離。

## Acceptance

3-tier AC 全 PASS で T-F-31 完了。本タスクは「セルフテスト」として、本 PR が gate #12
を実装した後、main の selected-stack.json の全項目 (deferred を除く) が PASS することを
確認する。
