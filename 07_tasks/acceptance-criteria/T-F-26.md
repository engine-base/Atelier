# T-F-26 Schemathesis contract test 統合

## Purpose

`07_api_design/openapi.yaml` を契約として、FastAPI 実装が契約に違反しないことを
Schemathesis で property-based に検証する独立 workflow を構築する。

v3-gate.yml の Gate #8 が **PR 時の同期 contract test** を担当するのに対し、
本タスクは:
- 単独の `contract-test.yml` workflow で nightly + on-demand 実行
- より長い examples 数 (200+) でランダム入力を試す
- 失敗時は GitHub Issue + PR comment で escalation

## Behavior

### scripts/run-schemathesis.sh
- uv で apps/api を install
- uvicorn で apps/api を boot (127.0.0.1:8765)
- 起動を待って schemathesis を実行
  - `--include-path-regex` で実装済 path のみ対象 (Phase 0 は /health)
  - `--hypothesis-max-examples=200` (Gate #8 より多い)
  - 全 checks: `not_a_server_error,status_code_conformance,content_type_conformance,response_schema_conformance,negative_data_rejection`
- 失敗時は exit 1

### .github/workflows/contract-test.yml
- triggers:
  - `pull_request` で `07_api_design/openapi.yaml` or `apps/api/**` 変更時
  - `schedule: cron '0 18 * * *'` (毎日 03:00 JST)
  - `workflow_dispatch` (手動)
- run `scripts/run-schemathesis.sh`
- 失敗時に GitHub Issue 自動起票 (label `contract-test-fail`)

## Acceptance

3-tier AC 全 PASS。
- workflow が PR / schedule / dispatch すべての trigger で起動
- 実装済 endpoint で 200 examples が回り PASS
- 未実装 endpoint は `--include-path-regex` で除外

## Failure modes

| 状態 | 挙動 |
|---|---|
| openapi.yaml と実装の drift | schemathesis が 5xx 検出 → workflow FAIL → Issue 起票 |
| 起動失敗 | curl ヘルスチェックがタイムアウト → FAIL |
| 認証必要 endpoint | `--auth` で test user token を渡す (Phase 0 未配線) |
