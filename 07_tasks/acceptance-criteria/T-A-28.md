X-Bridge-Token 認証で、Bridge worker からのみアクセス可能。**MIT 帰属表記必須**。

### kanban_complete のスコア振分ロジック

```python
async def complete_task(task_id: UUID, summary: str, metadata: dict):
    score = metadata['score']
    retry_count = metadata.get('retry_count', 0)

    if score >= 0.95:
        new_stage = 'done'
        approval_required = False
    elif score >= 0.80:
        new_stage = 'awaiting'
        approval_required = True
        await create_approval_inbox_entry(task_id, category='task_approval', score=score)
    elif retry_count < 3:
        new_stage = 'triage'  # 自動再試行
        await schedule_retry(task_id, retry_count + 1)
    else:
        new_stage = 'blocked'
        await create_approval_inbox_entry(task_id, category='blocked', urgent=True,
                                          reason='max_retries_exceeded')

    await update_task_lifecycle(task_id, new_stage, summary, metadata)
    await audit_log('task.complete', task_id, before=..., after=new_stage)
    return {"task_id": task_id, "new_lifecycle_stage": new_stage,
            "approval_required": approval_required}
```

## Tier 2: Functional（EARS 5 形式）

- [ ] **AC-F1**: EVENT-DRIVEN: When kanban_show is called with valid X-Bridge-Token, the system shall return active task info (title, description, AC, dependencies, blocked_reason if any).
- [ ] **AC-F2**: EVENT-DRIVEN: When kanban_complete with score >= 0.95, the system shall transition task to done, record summary+metadata in tasks table, and create audit_log.
- [ ] **AC-F3**: EVENT-DRIVEN: When kanban_complete with 0.80 <= score < 0.95, the system shall transition to awaiting, create approval_inbox entry with category=task_approval and score.
- [ ] **AC-F4**: EVENT-DRIVEN: When kanban_complete with score < 0.80 and retry_count < 3, the system shall transition to triage, increment retry_count, schedule retry within 5 min.
- [ ] **AC-F5**: EVENT-DRIVEN: When retry_count reaches 3, the system shall transition to blocked with reason='max_retries_exceeded' and create urgent approval_inbox entry.
- [ ] **AC-F6**: EVENT-DRIVEN: When kanban_block(reason) is called, the system shall transition to blocked, record blocked_reason, create urgent approval_inbox entry.
- [ ] **AC-F7**: EVENT-DRIVEN: When kanban_unblock is called by orchestrator role, the system shall transition from blocked to triage (re-try cycle restarts).
- [ ] **AC-F8**: STATE-DRIVEN: While Bridge worker is dead (PID polling fails for 60s), circuit_breaker shall mark dispatch_status=dead and re-spawn.
- [ ] **AC-F9**: UNWANTED: If X-Bridge-Token is invalid or expired, the system shall return 401 with code=INVALID_BRIDGE_TOKEN.
- [ ] **AC-F10**: UNWANTED: If task_id in request doesn't match Bridge worker's assigned task (HERMES_KANBAN_TASK env), the system shall return 403.
- [ ] **AC-F11**: UBIQUITOUS: All 7 tools shall record audit_log with actor_type=`ai_employee` and actor_id=task's assigned_employee_id.
