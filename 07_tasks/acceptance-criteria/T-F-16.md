# T-F-16 shadcn/ui + assistant-ui + tool-ui

## Purpose

selected-stack 確定の `ui_core: shadcn/ui + Tailwind` / `ui_ai_chat: assistant-ui` /
`ui_tool_calls: tool-ui` / `validation_ts: Zod` を本配線する。

## Behavior

### shadcn/ui
- `apps/web/components.json` で shadcn config 定義
- `apps/web/lib/cn.ts` で clsx + tailwind-merge の wrapper (cn 関数)
- `apps/web/components/ui/{button,card}.tsx` を最低 2 件配置
- apps/web/app/globals.css に shadcn theme variable を追加

### assistant-ui
- `@assistant-ui/react` を deps
- `apps/web/components/ai/thread.tsx` を Thread コンポーネント雛形として配置

### tool-ui (Lambda Curry/tool-ui)
- `tool-ui` を deps
- `apps/web/components/ai/tool-card.tsx` を ToolCall 表示の雛形として配置

### Zod
- `apps/web/package.json` (or root) に `zod` 追加
- `packages/shared/src/schema.ts` に共通 schema を配置 (Atelier の Project Status enum 等)
- `packages/shared/src/index.ts` から re-export

### tsconfig paths
- `apps/web/tsconfig.json` に `@/components/*`, `@/lib/*` の paths alias を追加

## Acceptance

3-tier AC 全 PASS。特に:
- Gate #1 (ESLint) で apps/web の新規 .tsx が strict pass
- Gate #3 (tsc) で 0-error
- Gate #4 (coverage) で touched ts ファイルが 80% 以上 (smoke unit test 必要)
- Gate #12 (selected-stack) で `validation_ts: Zod` check rule を re-enable して PASS

## Failure modes

| 状態 | 挙動 |
|---|---|
| zod 未配線 | Gate #12 validation_ts FAIL |
| shadcn cn 未配置 | components/ui/button.tsx が cn を import できず Gate #3 FAIL |
| 新規 tsx に test 無し | Gate #4 coverage 80% 未達で FAIL |
