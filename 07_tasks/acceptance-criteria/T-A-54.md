# T-A-54 — 受入基準

WS作成時のAI社員自動シード + 運営seed適用（製品ギャップ#27解消）。
本番実機検証（prod-smoke PS-22）で「新規WSにAI社員0・作成API無し→チャット開始不能」と判明。
根因は運営seed（ai_employee_templates）がdeployで流れず本番テンプレ0（INFRA-3と同型）。

## Tier 1 構造
- workspaces INSERT時に is_active テンプレから AI社員を実体化する SECURITY DEFINER トリガが存在する
  （owner-membershipトリガと同じブートストラップパターン）。
- deploy は migration 適用後に運営 seed（supabase/seed/*.sql）を冪等適用する。

## Tier 2 機能
- WS作成で、アクティブテンプレ1件につき ai_employee 1件を実体化（template_id/name/role/department 継承・is_default=true）。
- アクティブテンプレ0件でも社員0で例外を出さない（致命）。

## Tier 3 回帰
- 既存 workspace CRUD/RLS テストが従来どおり green。
- トリガ・seed は再適用で冪等（create or replace + 存在チェック / UPSERT）。

## 本番適用
- deploy に apply-seeds ステップを追加済。PROD_DATABASE_URL secret 経由で本番テンプレを投入し、
  以降の新規WSがトリガで固定10名を得る。既存WSは遡及シードされない（新規から有効）。
