# T-F-18 監査ログ書込ライブラリ（共通）

## Purpose

Atelier の全 API 操作・admin 操作を audit_log table に記録する共通ライブラリ。
- 誰が (actor_id)
- 何を (action / resource_type / resource_id)
- いつ (created_at)
- どこから (ip / user_agent)
- 結果 (status / metadata)

audit_log table 自体は T-D-01 系 (Group D) で migration 配置される想定。
本タスクは「書込側ライブラリ」を提供する。

## Behavior

### apps/api/src/audit/__init__.py
公開 API:
- `AuditEvent` dataclass
- `AuditWriter` クラス (insert ヘルパ)
- `AuditMiddleware` (FastAPI middleware)

### apps/api/src/audit/writer.py
- `AuditEvent` (frozen dataclass):
  actor_id, action, resource_type, resource_id, metadata, ip, user_agent, status_code
- `AuditWriter`:
  - `async write(event)`: SQLAlchemy AsyncSession 経由で INSERT
  - audit_log table schema が無くても import 可能 (実行時に動的 INSERT)
  - 失敗時は exception を伝播せず logger.warning に流す
    (audit 失敗で business request が落ちないよう defensive)

### apps/api/src/audit/middleware.py
- FastAPI ASGI middleware
- Request 受信時に actor_id (JWT claim) / ip / user_agent を抽出
- Response 後に status_code / action を確定して AuditWriter.write を呼ぶ
- 内部 endpoint (/health 等) は exempt

## Acceptance

3-tier AC 全 PASS。
- 80% coverage on touched files
- pyright strict: 0 errors
- Audit 失敗 が業務 request に影響しないこと (unit test で検証)

## Failure modes

| 状態 | 挙動 |
|---|---|
| audit_log table 未配置 (T-D-01 前) | INSERT failure → logger.warning, request は成功 |
| AsyncSession 取得失敗 | logger.warning, request は成功 |
| ASGI middleware 例外 | response は通常通り返し、例外は warn |
