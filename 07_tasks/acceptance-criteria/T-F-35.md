# T-F-35 shellcheck CI 統合

## Purpose

`selected-stack.json#shellcheck = "shellcheck"` に従い、shell script を CI で
shellcheck で lint する。bug を本番前に検出する。

## Scope

対象: `09_dispatch/scripts/*.sh` + `.husky/*` + その他リポジトリ内の `*.sh`

## Behavior

Gate #1 lint job 内に shellcheck step を追加:
- `sudo apt-get install -y shellcheck`
- `find . -name '*.sh' -not -path './node_modules/*' -not -path './.git/*' | xargs shellcheck --severity=warning`
- warning も error も 0 件が必須

## Acceptance

3-tier AC 全 PASS。Gate #12 selected-stack coverage check rule
(`shellcheck: v3-gate.yml で言及`) も PASS する。
