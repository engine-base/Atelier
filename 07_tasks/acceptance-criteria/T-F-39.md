# T-F-39: Better Stack ログ集約統合

## Purpose
T-F-08 (観測基盤) の Better Stack 部分を follow-up タスクとして分離。
構造化ログを Better Stack (https://betterstack.com) に集約し、Uptime 監視と
連動させる。

## Tier 1 — Structural
- `apps/api/src/observability/betterstack.py` が importable
- Python 標準 `logging` モジュールの Handler として実装 (`BetterStackHandler`)
- `apps/web/lib/logger.ts` が importable で `sendLog(level, message, context)` を提供
- 環境変数 `BETTERSTACK_SOURCE_TOKEN_API` / `BETTERSTACK_SOURCE_TOKEN_WEB` を読む

## Tier 2 — Functional (EARS)
- UBIQUITOUS: tsc + pyright strict 0-error
- EVENT-DRIVEN: When source token is missing, handlers shall fall back to local logging without raising
- UBIQUITOUS: 既存 Python `logging` の root logger に attach 可能

## Tier 3 — Regression
v3-gate.yml の 13 gate 全 PASS。
