# T-A-51 — 受入基準

web_search を chat SSE 実経路へ配線（T-F-21 消費側実装）。
第10軸 QA（AI-010〜013）で「utility は完成しているが本番呼び出し元ゼロ」と判明した起票漏れの解消。

## Tier 1 構造
- chat SSE の実 LLM stream（`_real_stream_chunks`）は `src/tools/web_search.py` の
  `build_web_search_tool()` が返す descriptor を `tools=` に注入する。独自 dict の直書き禁止。
- `ATELIER_WEB_SEARCH_DISABLED=1` で tools 注入を無効化できる（既定は有効）。

## Tier 2 機能
- assistant が streaming 中に web_search を使っても SSE イベント列
  （context → start → delta* → end）は well-formed のまま。
- `ATELIER_WEB_SEARCH_DISABLED=1` のとき provider へ tools を渡さない。
- provider が tool 関連エラーを返した場合、ユーザー向け定型エラーイベントを出し、
  生のプロバイダーエラーをクライアントへ流さない（致命 / バグ#17 の再発防止）。

## Tier 3 回帰
- 既存 chat SSE テスト（fake 経路含む）が従来どおり green。
- fake LLM 経路（`ATELIER_ALLOW_FAKE_LLM=1`）の挙動を変えない。
- 実プロバイダーでの tool 実走検証（正起動 / 誤選択防止 / max_uses）は
  `scripts/qa/ai_matrix_runner.py` の tool 行（AI-010〜013）で行う。
