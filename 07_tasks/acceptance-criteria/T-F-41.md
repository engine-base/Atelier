# T-F-41 — 受入基準

Bridge dispatcher クライアント実体。第10軸 AI-040/041 の解除タスク。
API 側（kanban 7 endpoints = T-F-28/T-A-28）は完成済みで、本タスクは apps/bridge の
クライアント側ループ（T-F-27 で「Vibeyard fork 取込後」とされていた実体）を headless 前提で実装する。

## Tier 1 構造
- dispatcher は API `/kanban/pick → start → complete | request-change` を `X-Bridge-Token` 認証で呼ぶ
  （DB 直叩き禁止・API 契約が唯一の経路）。
- headless エントリ（`node dist/headless.js`）で Electron UI 無しに 1 claim サイクルを実行できる。
- `claude -p`（print モード）を child_process で実行する。node-pty は導入しない（TUI 不要）。

## Tier 2 機能
- pick 成功時: `claude -p <タスクプロンプト>` を実行し、stdout/stderr を実行ログへ書き、exit 0 で complete を呼ぶ。
- exit 非 0 / timeout 時: request-change を呼び、タスクを retry 可能な状態に戻す。
- 実行中は一定間隔で heartbeat を送る。
- `ATELIER_BRIDGE_TOKEN` 不在 or 401 のとき、claim せずに停止する（致命）。

## Tier 3 回帰
- apps/bridge の tsc / vitest green。electron-entry の既存挙動不変。

## 実走検証（第10軸 evidence）
- 本物 API + 実 claude CLI で queued タスク 1 件を遂行し、
  task_executions / dispatch_status / S-I03 実行ログへの反映を evidence 化（AI-040）。
- 失敗系（exit 非 0）で request-change → 再 queue 可能を実証（AI-041）。
