# T-A-52 — 受入基準

Prompt Caching を chat SSE 実経路へ配線（T-F-15 消費側実装）。
第10軸 QA（AI-033）で「caching.py は完成しているが本番経路未使用」と判明した起票漏れの解消。
Atelier の system prompt は「ペルソナ + 装着スキル + プロジェクト状態」で毎ターンほぼ同一のため、
provider prompt cache の効果が最も出る形（コスト直結）。

## Tier 1 構造
- chat SSE の実 LLM stream は system prompt を `src/llm/caching.py` の
  `cache_system_prompt()` 経由の blocks（`cache_control` 付与）で渡す。
- `ATELIER_PROMPT_CACHE_DISABLED=1` で cache_control 付与を無効化できる（既定は有効）。

## Tier 2 機能
- 実 LLM 送信時、system は cacheable blocks として渡され、連続ターンで provider の
  prompt cache にヒットし得る形になっている。
- `ATELIER_PROMPT_CACHE_DISABLED=1` のとき system は plain string のまま。
- キャッシュ有効時も system prompt の意味内容を変えない（blocks 連結 = 原文一致）（致命）。

## Tier 3 回帰
- 既存 chat SSE テスト（fake 経路含む）が従来どおり green。
- fake LLM 経路の挙動を変えない。
