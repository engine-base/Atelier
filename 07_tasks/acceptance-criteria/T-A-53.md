# T-A-53 — 受入基準

daily_digest 実体実装 + Inngest serve 配線。第10軸 AI-042（cron 自律実行）の解除タスク。
T-F-20/T-A-40 は設計どおり Phase 0 skeleton（logger のみ）で完了しており、本タスクが「実体は別 task」の別 task。

## Tier 1 構造
- `main.py` は `ATELIER_INNGEST_ENABLED=1` のとき inngest の FastAPI serve を mount する（既定 OFF・既存挙動不変）。
- daily-digest handler は skeleton でなく `services/cron/digest.py` の実体を呼ぶ。

## Tier 2 機能
- 発火時、`cron_schedules` の enabled かつ `target_action='daily_digest'` の各 project について、
  タスク集計（lifecycle_stage 別件数）+ phase 状況 + 直近 24h の実行結果を集約した markdown digest を生成し、
  その project の「日次ダイジェスト」chat thread（無ければ作成）へ assistant message として保存、audit_logs に記録。
- 同一日付の digest message が既にある project はスキップ（冪等）。
- 対象 0 件でも例外にせず ok を返す（致命）。

## Tier 3 回帰
- 既存 `test_cron_scheduler.py` green・`ATELIER_INNGEST_ENABLED` 未設定時の挙動不変。

  DB 反映（chat_messages 追加）を `apps/web/.qa/evidence/ai-matrix/` に記録する。
