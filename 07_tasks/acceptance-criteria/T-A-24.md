## 実装仕様

### Input (PlayTaskRequest)
```typescript
{
  force?: boolean  // default false, true で blocked → in_progress を許可
}
```

### Processing
1. RLS で `tasks:member_update` policy 検証（403 if NG）
2. `tasks.lifecycle_stage` が `ready` または（`force=true` の場合）`blocked` であることを検証
3. `prerequisites[]` すべてが `done` であることを検証
4. 並列実行枠（10）の空きを確認 → 満杯なら queue に enqueue
5. `task_executions` レコード作成（status='running'）
6. tasks を `lifecycle_stage='in_progress', dispatch_status='spawning'` に更新
7. Bridge へ WebSocket イベント `spawn_worker` 送信（task_id, profile, worktree_path）
8. audit_logs に `task.play` イベント記録

### Output (PlayTaskResponse, 202 Accepted)
```typescript
{
  task_id: UUID,
  lifecycle_stage: 'in_progress' | 'queued',
  dispatch_status: 'spawning' | 'queued',
  execution_id: UUID,
  worktree_path: string,
  bridge_command: string,
  queue_position?: number  // 並列枠超過時
}
```

## Tier 2: Functional（EARS 5 形式）

- [ ] **AC-F1**: EVENT-DRIVEN: When task in lifecycle_stage=ready and play is requested with valid JWT, the system shall transition to in_progress, create execution record, emit spawn_worker event to Bridge via WebSocket, and return 202 with execution_id.
- [ ] **AC-F2**: EVENT-DRIVEN: When all prerequisite tasks complete, the system shall auto-promote dependent tasks from triage to ready (separate cron job).
- [ ] **AC-F3**: STATE-DRIVEN: While parallel slots are exhausted (10/10 in_progress), the system shall enqueue tasks and return 202 with queue_position >= 1.
- [ ] **AC-F4**: STATE-DRIVEN: While Bridge is offline (no heartbeat in 60s), the system shall return 503 with code=BRIDGE_OFFLINE.
- [ ] **AC-F5**: UNWANTED: If task lifecycle_stage is not in (ready, blocked with force=true), the system shall return 409 with code=INVALID_LIFECYCLE_STAGE.
- [ ] **AC-F6**: UNWANTED: If any prerequisite task is not done, the system shall return 409 with code=DEPENDENCIES_NOT_MET and details listing unmet dependencies.
- [ ] **AC-F7**: UNWANTED: If user role is viewer, the system shall return 403 with code=FORBIDDEN.
- [ ] **AC-F8**: UBIQUITOUS: The system shall enforce row-level access via `tasks:member_update` policy (RLS).
- [ ] **AC-F9**: EVENT-DRIVEN: When play succeeds, the system shall record audit_log with action=`task.play`, target_type=`task`, target_id, before/after lifecycle_stage.
- [ ] **AC-F10**: UNWANTED: If task is rate-limited (10/min/user exceeded), the system shall return 429 with code=RATE_LIMITED.
