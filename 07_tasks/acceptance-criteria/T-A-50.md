# T-A-50 — 受入基準

運営ナレッジ管理API(F-023)：platform ナレッジ CRUD + visible_in_tree (admin + service_role)

## Tier 1 構造
- UBIQUITOUS: platform(運営デフォルト)ナレッジの作成/編集/削除/一覧 と visible_in_tree トグル を openapi 通りに /admin/knowledge で公開する
- UBIQUITOUS: 運営ナレッジ管理は管理者(admin)専用で service_role により RLS をバイパスして書込む

## Tier 2 機能
- [EVENT-DRIVEN] When an admin creates operator-default knowledge, the system shall store it with account_type=platform and a fixed sentinel account_id
- [EVENT-DRIVEN] When an admin toggles visible_in_tree, the system shall persist the flag and a fresh GET shall reflect it
- [UNWANTED] If a non-admin calls operator-knowledge management endpoints, the system shall return 403 Forbidden （致命）
- [UNWANTED] If the request is unauthenticated, the system shall return 401 Unauthorized （致命）

## Tier 3 回帰
- 既存のテナント knowledge CRUD (RLS / /knowledge) が従来どおり動作する
