# T-F-33 uv (Astral) Python パッケージマネージャ本配線

## Purpose

`03_architecture/selected-stack.json#package_manager_py = "uv (Astral)"` の確定通りに
uv を本配線する。pip を撤回し、uv sync / uv run / uv pip install で全 Python deps
管理を行う。

## Behavior

### Root pyproject.toml (T-F-02 設置)
- `[tool.uv.workspace.members = ["apps/api"]]` を維持
- `[tool.uv]` セクションに `package = false` を維持 (root はパッケージではない)

### apps/api/pyproject.toml
- `[project.optional-dependencies].dev` をそのまま維持
- `uv sync --all-extras` で dev deps を含めて install できる状態

### uv.lock
- root に uv.lock を生成して commit する
- 全 dep の lock pin で再現性を保証

### v3-gate.yml
- pyright install / apps/api install を `uv sync` ベースに置き換え
- pytest 実行を `uv run pytest` に置き換え
- schemathesis 実行も `uv run` 経由

## Acceptance

3-tier AC 全 PASS。
特に **Gate #12 selected-stack coverage** で `package_manager_py = uv` が PASS する
ことを保証する (uv.lock or [tool.uv] の存在検出)。

## Failure modes

| 状態 | 挙動 |
|---|---|
| uv.lock 無し | Gate #12 check rule で `[tool.uv` も無ければ FAIL |
| pip 使用 | CI step で `pip install -e ...` が残っているなら本 PR で除去 |
| uv sync 失敗 | gate #4 fail (pytest が動かない) |
