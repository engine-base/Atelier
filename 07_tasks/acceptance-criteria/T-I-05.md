## 試験スコープ

### Layer 1: SQL レベル（最も厳密・データベース直接）

複数 workspace の seed を用意し、Supabase Postgres に直接接続して RLS が正しく動くかを SQL で検証：

```sql
-- セットアップ
INSERT INTO workspaces (id, name) VALUES ('ws_A', 'Workspace A'), ('ws_B', 'Workspace B');
INSERT INTO users (id, email) VALUES ('user_A', 'a@test.com'), ('user_B', 'b@test.com');
INSERT INTO workspace_memberships VALUES ('user_A', 'ws_A', 'owner'), ('user_B', 'ws_B', 'owner');
INSERT INTO projects VALUES ('proj_A', 'ws_A', 'Project A'), ('proj_B', 'ws_B', 'Project B');

-- 越境試行
SET request.jwt.claims = '{"sub": "user_A", "role": "authenticated"}';
SELECT * FROM projects WHERE workspace_id = 'ws_B';  -- 期待: 0 件
SELECT * FROM tasks WHERE project_id = 'proj_B';     -- 期待: 0 件
```

### Layer 2: API レベル

実 FastAPI を立てて、user_A の JWT で user_B のリソースに HTTP リクエスト：

```python
# user_A としてサインイン
token_a = sign_in("a@test.com", "password")

# user_B のプロジェクトに直接 GET
resp = client.get(f"/api/v1/projects/{proj_B.id}", headers={"Authorization": f"Bearer {token_a}"})
assert resp.status_code == 404  # NOT 200, NOT 403（存在開示禁止）

# user_B のタスクを再生試行
resp = client.post(f"/api/v1/tasks/{task_B.id}/play", headers={"Authorization": f"Bearer {token_a}"})
assert resp.status_code == 404
```

### Layer 3: Client Portal 別 JWT 経路（R-T08 致命級・絶対分離）

クライアント招待 JWT は **`role=client_portal` + `project_id` クレームのみ**。users / workspaces / 他 projects に絶対アクセス禁止：

```python
# クライアントサインイン → JWT 取得
client_jwt = client_signin(invitation_token)  # role=client_portal, project_id=proj_A

# クライアントが users / workspaces を取得試行
resp = client.get("/api/v1/auth/me", headers={"Authorization": f"Bearer {client_jwt}"})
assert resp.status_code in (401, 403)

resp = client.get(f"/api/v1/workspaces", headers={"Authorization": f"Bearer {client_jwt}"})
assert resp.status_code in (401, 403)

# 別 project_B にアクセス試行
resp = client.get(f"/api/v1/client/projects/{proj_B.id}", headers={"Authorization": f"Bearer {client_jwt}"})
assert resp.status_code == 403

# 自分の project_A は OK
resp = client.get(f"/api/v1/client/projects/{proj_A.id}", headers={"Authorization": f"Bearer {client_jwt}"})
assert resp.status_code == 200
```

### Layer 4: UI レベル（Playwright）

ブラウザで user_A としてログイン → URL 直叩きで他 WS のページにアクセス → リダイレクト or 404 確認。
