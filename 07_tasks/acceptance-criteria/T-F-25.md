# T-F-25 OpenAPI → TS 型 / Pydantic 自動生成パイプライン

## Purpose

`07_api_design/openapi.yaml` を信頼源として、TS / Pydantic の型を機械生成する
パイプラインを構築する。

これにより:
- BE (FastAPI Pydantic) と FE (apps/web TanStack Query 等) の型が常に同期
- Gate #7 type-drift が実体実装される (即時的に drift 検出)

## Behavior

### scripts/sync-types.sh
- npx openapi-typescript で `packages/api-types/src/openapi.ts` を生成
- uv tool で datamodel-code-generator を実行し `apps/api/src/_generated/openapi_models.py` を生成
- 両方の生成物を git diff で確認 (drift 検出)

### packages/api-types/
- `@atelier/api-types` workspace
- `src/openapi.ts` = openapi-typescript 出力 (paths / components / operations 型)
- apps/web から `import type { paths } from '@atelier/api-types'` で参照

### apps/api/src/_generated/
- `__init__.py` = generated marker
- `openapi_models.py` = datamodel-code-generator 出力 (Pydantic v2 models)
- apps/api のコードから `from src._generated.openapi_models import ...` で参照

## Acceptance

3-tier AC 全 PASS。
- Gate #7 type-drift で `packages/api-types/src/openapi.ts` の再生成結果が committed
  ファイルと一致 (drift 0)
- scripts/sync-types.sh が冪等に動作
- 生成物は手動編集禁止 (header コメントで明示)

## Failure modes

| 状態 | 挙動 |
|---|---|
| sync-types.sh 実行失敗 | npx / uv tool が解決できず Gate fail |
| openapi.yaml と生成物の drift | Gate #7 で diff 検出 → FAIL |
| 手動編集 | sync-types.sh 再実行で上書きされる |
