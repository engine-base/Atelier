# T-F-32 CI Gate #13 — gap tracker

## Purpose

PR description / commit messages / diff の中の **gap マーカー** を検出して
GitHub Issue を自動起票する。「あとで」「TODO」「placeholder」を口にした瞬間に
gap が記録され、closure まで追跡される状態を作る。

## Behavior

1. PR description と commit messages を取得
2. 以下のパターンを検出:
   - `_TRACK:` 接頭辞 (意図的に gap として登録するマーカー)
   - 新規追加された diff 内の `TODO` / `FIXME` / `XXX` コメント
3. 検出した gap ごとに GitHub Issue を起票 (ラベル `gap-tracker,follow-up`)
4. PR にリンクを comment

`_TRACK:` だけで起票、`TODO/FIXME` は warning + 起票 (FAIL ではない)。

## Acceptance

3-tier AC 全 PASS。
test scenario として PR description に `_TRACK: テスト` と書くと
対応する Issue が起票されることを確認。
