# T-A-47 — 受入基準

ナレッジ拡張API：project scope + 構造ツリー(parent_id) + 運営デフォルト(platform)対応

## Tier 1 構造
- UBIQUITOUS: The system shall accept scope=project (requiring source_project_id), parent_id (tree), visible_in_tree, account_type=platform as defined in 07_api_design/openapi.yaml
- UBIQUITOUS: 一覧/ツリーは visible_in_tree=true のみ返し、検索(RAG)は visible_in_tree に関わらず platform を含める

## Tier 2 機能
- [EVENT-DRIVEN] When a member creates knowledge with scope=project, the system shall persist source_project_id and return it
- [EVENT-DRIVEN] When listing with parent_id=X, the system shall return only direct children of X
- [EVENT-DRIVEN] When any tenant's AI searches, the system shall include account_type=platform knowledge in cosine results
- [UNWANTED] If a tenant member attempts to create/update/delete account_type=platform knowledge, the system shall reject (403/RLS) — write は service_role のみ （致命）
- [UNWANTED] If the request is unauthenticated, the system shall return 401 Unauthorized （致命）

## Tier 3 回帰
- 既存 scope=common / employee_specific のナレッジ CRUD と Voyage 埋め込み/cosine 検索が従来どおり動作する
- VOYAGE_API_KEY 未設定でも None フォールバックで落ちない
