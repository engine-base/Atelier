# T-F-10 Drizzle ORM セットアップ

## Purpose

`selected-stack.json#orm_ts = "Drizzle ORM"` に従い TS 側の ORM 基盤を配線する。

migration は **Supabase CLI 一本化** が方針 (selected-stack#orm_ts.review)。
drizzle-kit は migration には使わず schema 型生成 + studio 用とする。

## Behavior

### packages/db/package.json
- `drizzle-orm` + `drizzle-kit` を deps
- workspace package `@atelier/db` として配置
- scripts: `drizzle-kit studio` / `drizzle-kit generate` (migration 確認用)

### packages/db/drizzle.config.ts
- dialect: postgresql
- schema: ./src/schema/*.ts
- out: ./drizzle/migrations
- driver: pg (Supabase Postgres)
- DATABASE_URL を環境変数から読む

### packages/db/src/schema/.gitkeep
schema 定義は T-D-01〜30 (Group D) で追加される。本タスクは empty dir のみ確保。

## Acceptance

3-tier AC 全 PASS。
- Gate #12 selected-stack で orm_ts が新規 check rule で検出可能
- packages/db が pnpm workspace から認識される
- drizzle-kit --help が動く

## Failure modes

| 状態 | 挙動 |
|---|---|
| drizzle-orm 未 install | Gate #3 (tsc) で import 解決不可 |
| drizzle.config.ts 構文不正 | drizzle-kit が動かず手動確認で発覚 |
