# T-F-20 Inngest cron 基盤

## Purpose

`selected-stack.json#queue = "Inngest (Phase 4+) + pg_cron (Phase 0-3)"` の通り
Inngest を cron / async job のプラットフォームとして本配線する。T-F-13 の
inngest client 単独設定を基に、**cron function 登録のパターン**と
**scheduler 起点**を提供する。

## Behavior

### apps/api/src/cron/__init__.py
公開 API:
- `register_cron_jobs` — Inngest client に全 cron function を登録
- `CronSchedule` — cron 定義のメタデータ型

### apps/api/src/cron/scheduler.py
- `CronSchedule` dataclass: `name`, `cron`, `description`
- `register_cron_jobs(client)`: scheduler 起点。
  inngest_handlers の各 handler を client に登録する
- Atelier の例示 cron (実体は別 task で実装):
  - `daily-digest` (毎朝 7:00 JST = 22:00 UTC)
  - `weekly-burndown` (毎週月曜 9:00 JST)

### apps/api/src/cron/inngest_handlers.py
- Inngest `@create_function` を使った cron handler の雛形
- Phase 0 では `daily_digest` / `weekly_burndown` 2 件の skeleton 関数
  (実 body は別 task / 別 sprint で実装)
- 各 handler は Inngest event を受け取り、AsyncSession で DB 書き込み or 通知

## Failure modes

| 状態 | 挙動 |
|---|---|
| Inngest client 未初期化 | register 時に例外 → 起動時 fail |
| cron 式不正 | inngest 側で reject |
| handler 実装無し | skeleton 関数が None を返す (期待通り) |
