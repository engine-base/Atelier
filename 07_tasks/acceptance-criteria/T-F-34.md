# T-F-34 Husky + lint-staged 配線

## Purpose

`selected-stack.json#git_hooks = "Husky + lint-staged"` に従い、commit 前に
staged file の lint / format / type-check を自動実行する状態を作る。

## Behavior

### .husky/pre-commit
- `pnpm exec lint-staged` を実行
- staged file に対して lint-staged の rule を適用

### .husky/commit-msg
- conventional commits 形式の最低限チェック
  (例: `^(feat|fix|chore|docs|refactor|test|perf|ci|build|style|revert)(\(.+\))?: `)

### .lintstagedrc.json
- `*.{ts,tsx,js,jsx}` → `pnpm -w next lint --fix --file` または `prettier --write`
- `*.py` → `uv tool run ruff check --fix` + `uv tool run ruff format`
- `*.{json,md,yml,yaml}` → `prettier --write`

### package.json
- devDependencies に `husky` + `lint-staged` を追加
- `prepare` script に `husky` を追加 (post-install で hook を install)

## Acceptance

3-tier AC 全 PASS。特に Gate #12 selected-stack coverage で
`git_hooks` check rule (`.husky/pre-commit 存在`) が PASS する。
