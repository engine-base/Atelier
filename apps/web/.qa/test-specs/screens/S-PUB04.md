## design-audit ラウンド v2 (2026-07-19) — 4 画面共通の是正

> モック: `06_mockups/public/S-PUB0*.html` / 実装: `apps/web/app/public/`
> 実操作: `.audit-pub.mjs` 14/14 PASS

### 検出した主要欠陥
1. **偽フォーム (S-PUB04)**: onSubmit が no-op — 削除申請しても何も起きなかった。実 API
   (POST /public/data-deletion-requests) が存在するのに未配線 → 配線し、audit_logs 記録まで実証。
2. **法令文書の正本乖離 (S-PUB01〜03)**: GET /public/legal-documents の正本 (DB) が存在するのに
   各ページがハードコード縮約版を表示 — 特商法は事業者名・連絡先まで正本と異なっていた
   (法令ページとして重大)。正本描画方式 (LegalDocArticle) に統一。
3. 公開ヘッダー (ブランド + 法令ナビ) がモックにあるのに全ページ欠落 → PublicHeader 新設。
4. 見出し二重表示 (body_md 先頭 H1 とページ h1) を実走スクショ目視で検出 → 除去。

### 教訓
- 法令系ページは「静的でよい」ではなく**正本がどこかを先に確認する** — 正本 API があるのに
  ハードコードすると、改訂時に画面だけ古いままになる (特商法の事業者名乖離が実例)。
