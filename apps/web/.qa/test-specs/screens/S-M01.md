### 検出・修正した欠陥

1. **D&D 未実装** (「ここにファイルをドロップ」と表記しながらドロップ不可)
2. **accept 属性がテキストを弾く** (.txt/.docx チップ表示と矛盾)
3. **アップロード履歴が皆無** (アップロードするたび過去分が見えない)
4. page に QueryProvider 欠落 (今回の react-query 化で顕在化) / 5. 開く失敗時のエラーが空文言

### 追記 (2026-07-19 深掘り): パイプラインは UI 契約より深い所で断絶している

ユーザー指摘を受けた再調査で判明: **Whisper worker がリポジトリのどこにも存在しない** (GAP-016)。
POST /transcribe は DB に queued と書くだけで、`transcripts/queued/` を消費するコードが無い。
つまり storage を設定した本番相当でも parsed_at は立たず、アップロード → 文字起こしは必ず
タイムアウトする。当初この監査は「storage 未設定が dev の制約」とだけ記載しており、
worker 不在という真因を見抜けていなかった (unit テストは container の polling 契約しか
検証していない)。教訓: 「キュー登録 API がある」ことと「キューを処理する者がいる」ことは
別に検証する — 非同期パイプは終端まで所有者を確認する。
