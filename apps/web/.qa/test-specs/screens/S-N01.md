### モック要素台帳 (Rule 10 全分類)

| モック要素 | 分類 | 対応 |
|---|---|---|
| (導線) | **実装** | **到達不能画面だった** — ナビに「営業ドラフト」を新設 (ConditionalAppShell) |
| 種別タブ 5 種 | 実装 2 + 撤去 3 | 提案書/見積書 = API doc_type と一致する実タブ (実件数バッジ)。業務委託契約/NDA/請求書 = doc_type 未対応で撤去 + GAP-018 |
| doc-toolbar 修正依頼 | **実装** | disabled placeholder → /chat への実リンク |
| doc-toolbar PDF / 送信 | **撤去 + GAP-018** | disabled placeholder (Rule 10 違反) を全廃 |
| doc-toolbar 編集 | 実装 | PATCH /sales-docs/{id} (従来から) |
| ドキュメントプレビュー | 実装 | 版数バッジ v{n}・作成日は YYYY-MM-DD |
| 生成プロセスカード | 実装 (honest 化済) | 虚偽の実行トレース表示 → 汎用手順 (前ラウンド是正を維持) |

### 実装追加 (モックに無い改善)
- 保存済みドキュメント一覧 (GET /sales-docs、版数・日付・選択・2 段階削除) — 旧実装はリロードで作成物に二度と触れなかった
- 作成成功でフォームをクリアし、作成した版をプレビューに自動表示

### 教訓
- 「見た目はタブ・実体は span」の死にタブは、タブ切替 TC を**aria-selected の遷移**まで
  見ないと検出できない。
- disabled placeholder は Rule 10 違反として一掃する — 「非活性の視覚クローム」というコメント付き
  実装が監査をすり抜けていた (コメントで自覚していても違反は違反)。
