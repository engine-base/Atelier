## 主要な設計判断

### 仕様徹底ループ（F-J02）
- スコア計算：`AC達成率×0.6 + テスト成功率×0.3 + 検証AI評価×0.1`
- 段階制御：
  - ≥95% → 自動承認
  - 80-94% → 人間承認ゲート
  - <80% → 強制人間介入
- 最大再試行：3回
- スキル化：`verification-loop` `acceptance-criteria` `test-verification` `quality-gate` をClaude Code側で自走

### 並列実行エンジン（F-I03）
- ローカルClaude Codeブリッジ、5-10並列
- 初回はターミナルCLIログイン、以降自動起動
- バックグラウンド継続、画面閉じてもタスク進行
- ストリーミングはオプション（通常/モニターの2モード）
- 承認待ちは Inbox + メール通知

### /goal駆動実行（F-I05）
- タスクカード → ゴール文自動生成 → Claude Code /goal 投入
- 自走で実装→検証→修正→検証... ループ
- 自然言語からゴール抽出も可

### かんばん構造（F-I02）
- 縦軸：カテゴリ（機能/画面/フェーズ/担当別を切替可）
- 横軸：未着手 / 実装中 / 検証中 / 完了 / ブロック
- 詳細パネル：仕様/モック/受入基準/テスト/依存/API/ナレッジ参照/実行ログを全て閲覧

### ナレッジ構造（F-K01）
- 2層：共通 ＋ AI社員別
- 各社員別カテゴリ例：
  - ワンダ：参考デザイン / カラー傾向 / フォント選好
  - トニー：過去提案テンプレ / 成約パターン
  - ストレンジ：採用アーキ / 失敗パターン
  - ヴィジョン：仕様逸脱例 / 検証チェックリスト
- 各社員は自分のナレッジ優先＋共通も参照可
