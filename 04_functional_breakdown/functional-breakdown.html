1. エンティティ (Entities)

25 テーブル、合計 85 の RLS ポリシー定義。命名規約 PascalCase Entity → snake_case_plural table。全テーブルで UUID v7 + ソフトデリート + TIMESTAMPTZ を採用。

主要エンティティ抜粋

E-007decided
AiEmployee (table: ai_employees)

三階層組織のAI社員。デフォルト10名 + ユーザー雇用可。ユーザー編集可項目（display_name/icon/tone_preset/custom_tone_text）と管理側編集項目（system_prompt/attached_skills/role/department）を分離。

E-012decided
Task (table: tasks)

1-2日粒度。dependencies/prerequisites/blocks 必須属性。acceptance_criteria_id/mock_id/spec_html_path 必須リンク。files_changed で衝突回避。

E-018decided
KnowledgeNode (table: knowledge_nodes)

アカウント単位（workspace or user）の2層ナレッジ。scope = common または employee_specific。Voyage AI 1024次元 embedding + HNSW + FTS + tags GIN。

2. 機能 (Features)

合計 35 機能。decided 32 / draft 2 / blocked 1（F-LEGAL-014 電気通信事業届出は法務確認待ち）。すべて EARS 5 形式の AC seed を持つ。

注目機能

F-CTX01decided
Hybrid Context Construction Layer

7層コンテキスト構築：ペルソナ → 直近メッセージ → DB現在状態（DB-as-truth）→ セマンティック関連メッセージ → RAGナレッジ → mem0長期記憶 → 言及エンティティ → 必要時LLMLingua圧縮。OSS活用で自前は100行未満。

F-IMP01decidedPhase 3
Impact Analyzer（影響範囲解析）

NetworkX で tasks.dependencies DiGraph を構築、descendants() で影響タスク抽出、LangGraph で解析フロー。フェーズ移動/追加/仕様変更時に自動起動 + 日次cron。

F-018decidedPhase 3
並列実行エンジン（5-10並列）

ローカル Claude Code WebSocket/SSE ブリッジ。初回CLIログイン、BG継続、ストリーミングはオプション。スコア評価（F-021）+ 人間承認ゲート（F-022）と連携。
