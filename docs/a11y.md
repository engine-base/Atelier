# a11y ガイド (T-US-13 / WCAG 2.2 AA)

Atelier の Web UI は **WCAG 2.2 Level AA を最小基準**として実装する。本ドキュメントは
チーム共通の判断基準と、コードレベルで遵守すべき項目を定める。

## 信頼源

- WCAG 2.2: https://www.w3.org/TR/WCAG22/
- axe-core ルール: https://dequeuniversity.com/rules/axe/
- Atelier デザイントークン: `05_design/DESIGN-atelier.md` および `packages/design-tokens/src/index.ts`
- a11y ヘルパ: `apps/web/lib/a11y.ts`

## 必達ルール (PR で必ず満たす)

### 1. キーボード操作のみで完結

- 全ての対話要素は Tab / Shift+Tab で到達可能、Enter / Space で起動可能
- モーダル/ドロワー/メニュー開状態では **`createFocusTrap()` でフォーカスをトラップ**
- 閉じた直後は呼び出し元 (トリガ) にフォーカスを返す
- Esc で閉じられる UI は `aria-modal="true"` を付ける

### 2. コントラスト比

- 通常テキスト: **>= 4.5:1**
- 大テキスト (18pt / 14pt bold 以上): **>= 3.0:1**
- インタラクティブ要素の境界線 / icon: **>= 3.0:1**
- `meetsContrastAA(fg, bg, large?)` をテストで使うこと

### 3. フォーカスインジケーター

- `apps/web/styles/tokens.css` の `*:focus-visible` で `2px` の primary 色 outline を強制
- これを `outline: none` 等で無効化することは禁止
- カスタムコンポーネントは focus ring を追加で描いても構わないが、既定 outline を消さない

### 4. セマンティック HTML

- 見出しは `<h1>..<h6>` で階層化 (`<div>` の class 名で誤魔化さない)
- ナビゲーションは `<nav>`、メイン領域は `<main>`、補助情報は `<aside>`
- フォーム入力は `<label for="...">` を必ず関連付ける (placeholder のみ不可)
- 装飾的画像は `alt=""` で SR から隠す、情報画像は意味のある alt を付ける

### 5. 動的更新の通知

- 非モーダルな状態変化 (Save 完了 / 検索ヒット件数 等) は `announce()` で aria-live=polite 通知
- エラーは `aria-live=assertive` (緊急アナウンス) を使う

### 6. モーション尊重

- `prefers-reduced-motion: reduce` 環境ではアニメーションを 0.01ms に切る
- `apps/web/styles/tokens.css` でグローバル媒体クエリ実装済 — 個別 CSS で上書きしない

## コードレベル checklist (各コンポーネント実装時)

- [ ] フォーカス可能要素は keyboard で全到達確認 (`getFocusableElements()` で列挙)
- [ ] 配色は `meetsContrastAA()` を unit test で検証
- [ ] aria-label / aria-labelledby を必要に応じて付与
- [ ] disabled 状態は `aria-disabled` ではなく真の `disabled` を使う (フォーム要素)
- [ ] modal は role="dialog" + aria-modal="true" + aria-labelledby
- [ ] アイコンのみのボタンは `aria-label="操作名"` を付ける
- [ ] form field は label + error message (`aria-describedby`) を関連付け

## 自動テスト

- Vitest 単体テスト: `apps/web/tests/bundle-a/a11y.test.ts` でヘルパの正しさを検証
- E2E (T-I-10): Playwright + `@axe-core/playwright` で各画面の WCAG ルール違反 0 件を検証 (CI Gate)
- これらは **`should-pass` のみ計上**: dev/lint 段階で違反を検出した場合はその場で修正
