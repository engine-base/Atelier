3 案件で共通する認証フローのパターンを抽出。Magic Link 優先 + OAuth フォールバック + 段階的同意取得 の組合せが成約率・離脱率ともに最適だった。

共通パターン

  • Magic Link を主体：パスワード入力を避け、Resend 経由で OTP を配信。15 分有効。
  • OAuth は補助：Google + GitHub の 2 つのみ提供。多すぎると選択疲労が出る。
  • 同意は 1 つにまとめる：利用規約 / プライバシーポリシー / 越境同意 を 1 つのチェックボックスに集約。離脱率 18% → 6% に改善。
  • サインアップとサインインを 1 画面に：タブ切替で対応。新規・既存どちらも同じ画面から始められる。

避けるべきパターン

  • パスワード強度バーの早期表示（ユーザーを焦らせる）
  • 2FA の強制（個人プランでは OFF、エンタープライズでは推奨）
  • 3 つ以上の OAuth プロバイダー（選択肢が多すぎ）
